TypeScript path alias `@/` maps to `src/` (defined in `tsconfig.json`).

## Architecture

### Rendering Model
Astro renders all pages as static HTML at build time. Dynamic interactions are handled via vanilla client-side TypeScript in `<script>` blocks. Each major page (`index.astro`, `assessment.astro`) contains its own logic focused on its specific features.

### State & Data Flow
1. **Persistence:** Appliance configurations are stored in `localStorage` under `electricity_load_calculator_state`.
2. **Hydration:** On page load, the application hydrates the UI from `localStorage` if available.
3. **Reactivity:** All interactions (qty stepper, watts edit, preset load) mutate a local state object, which then triggers a UI re-render (DOM updates) and persists the change.
4. **Assessment:** The `assessment.astro` page retrieves the same state to generate deep-dive reports, health scores, and component recommendations.

### Component Logic
- **`src/utils/calculations.ts`**: Pure functions for basic load math.
- **`src/utils/health.ts`**: Logic for health scores (0-100), savings tips, and consumption alerts.
- **`src/utils/insights.ts`**: Engineering-grade sizing logic for MCBs, Cables, Inverters, and Solar.

### Critical Invariant: SSR/Client-Side Sync
Dynamic rows generated by `buildRowHTML()` (client-side) must strictly match the structure and attributes of `ApplianceRow.astro` (server-side) to ensure event delegation and styling consistency.

### Navigation & Breadcrumbs
The application uses a multi-page structure with breadcrumbs for deep routes. The `SiteHeader.astro` handles active link highlighting and mobile menu logic using native AbortController for clean listener cleanup.

### Knowledge Hub (`/knowledge-hub/**`)
A Phase 1-complete SEO content cluster. URL hierarchy:
```
/knowledge-hub                          → Hub Homepage
/knowledge-hub/load-calculation         → Cluster Landing
/knowledge-hub/load-calculation/[slug]  → 6 full articles
/knowledge-hub/mcb-sizing|wire-sizing|solar-inverter  → Phase 2 stubs
```
- Data registry: `src/data/knowledgeHub.ts`
- Layout: `src/layouts/KnowledgeHubLayout.astro` (TOC sidebar, breadcrumbs)
- Shared components: `ClusterCard.astro`, `QuickAnswerBox.astro`, `KnowledgeHubCTA.astro`
- Phase 2 (MCB Sizing, Wire Sizing, Solar & Inverter) is pending.

### Typography Conventions
- **Base font-size:** `html { font-size: clamp(16px, 1vw + 13px, 18px) }` — fluid 16→18px.
- **Body:** `line-height: 1.75; letter-spacing: 0.01em` — compensates for Hind's condensed glyphs.
- **Tailwind text scale** is bumped one step up in `@theme` (global.css) — `text-sm` = 15px, `text-base` = 16–18px fluid.
- Font roles: `font-display` = Khand (headings/UI), `font-body` = Hind (body text), `font-mono` = Geist Mono (data, FAQ answers).

### FAQ Answer Convention
- All FAQ answer `<p>` tags **must** use `.faq-answer` class (defined in `global.css`).
- Applies Geist Mono, `text-sm`, `line-height: 1.8` — calm and readable.
- **Never** use ad-hoc `text-sm text-text-muted leading-relaxed` on FAQ answers.
- Applies to: `FAQSection.astro`, all `recommendations/*.astro`, `knowledge-hub/index.astro`, `knowledge-hub/load-calculation/index.astro`, all Knowledge Hub articles.

## Testing

### End-to-End Tests (`tests/e2e/`)
- **`electricity-load-calculator.spec.ts`**: Focuses on core calculator interactions (steppers, custom adds, presets, theme toggling).
- **`site-pages.spec.ts`**: Verifies navigation across all pages (About, Contact, Privacy, Terms), 404 handling, and data persistence on the Assessment page.
- **Strict Mode:** Tests use specific selectors (e.g., `main h1`) to avoid conflicts with the Astro dev overlay.

### Logic Verification (`scripts/verify-logic.cjs`)
A standalone script for fast verification of calculation functions without requiring a browser or build step.

## Sitemap & Indexing Rules
- Filter is in `astro.config.mjs`. Currently **excludes** `/404` and `/assessment`.
- `/assessment` has `noindex={true}` AND is excluded from sitemap — Google rejected it (no meaningful content without calculator state).
- All `/knowledge-hub/**` pages are included by default (SEO content, no action needed).
- When adding new pages: only add to the exclusion filter if the page is dynamic/stateless.
